Canopy Guard - Free Website Visibility and Security Audit

Canopy Guard is a free website audit tool that scans any domain in 15 seconds and scores it across four layers: SEO, AEO, GEO, and Security — and five layers of security analysis: HTTP response, HTML content, DNS security depth, TLS depth, and path probing. It then cross-references visibility findings against security findings to surface gaps that no other tool catches independently. Built by Adam McClarin, CISSP, at Meraki is Love Digital (Soulful Tech), based in Friendswood, Texas.

What does Canopy Guard check?

Canopy Guard runs 17 scan modules in parallel against any domain, checking 83 signals across four scoring layers. Every scan returns live results from real HTTP requests. No simulations, no cached data. Average scan time is 5 to 15 seconds.

SEO: 14 Signals

The SEO layer measures how well search engines can crawl, index, and rank your site. It checks crawlability, H1 tag count (exactly one is ideal, multiple are penalized), H2 heading structure, meta description presence and length (120 to 160 characters is ideal), title tag presence and length (30 to 60 characters is ideal), canonical URL matching, mobile viewport meta tag, HTML lang attribute, image alt text coverage, word count with gradient scoring, internal link count, sitemap.xml existence, and page response time.

AEO: 10 Signals

AEO stands for Answer Engine Optimization. It measures how well AI answer engines like ChatGPT, Google AI Overview, and Perplexity can extract and cite your content. Canopy Guard checks for any JSON-LD structured data, Organization schema, FAQ schema presence, FAQ item count (5 or more items gets full credit), LocalBusiness or service schema, Breadcrumb schema, schema type diversity, JSON-LD block count, validation errors, and Q&A content density. Without proper schema markup, your site may be invisible to the AI tools that millions of people now use to find businesses and services.

GEO: 8 Signals

GEO stands for Generative Engine Optimization. It measures how well generative AI models can chunk, retrieve, and cite your pages in their responses. Canopy Guard checks chunking efficiency (how cleanly your content divides into approximately 350 token semantic blocks based on heading count, paragraph count, lists, and tables), citation precision (the ratio of specific verifiable data points like numbers, proper nouns, dates, and percentages to generic text), llms.txt file status and length, content depth by word count, presence of ordered or unordered lists, presence of tables, and heading to content ratio. Pages with well-structured content and specific claims get cited more often than pages with generic copy.

Security: 51 Signals

The Security layer measures the external security posture visible from outside the site. It checks TLS certificate validity, HSTS header with max-age scoring (1 year or more earns extra credit), HTTP to HTTPS redirect, Content-Security-Policy header (weighted highest at 0.08), Strict-Transport-Security header, X-Frame-Options header, X-Content-Type-Options header, Referrer-Policy header, Permissions-Policy header, cookie security flags (Secure, HttpOnly, SameSite), AI crawl policy classification (BALANCED scores higher than RESTRICTIVE or PERMISSIVE), AI bot awareness in robots.txt, rate limiting directives, exposed endpoint detection with false positive filtering, and vulnerability indicators like server version disclosure. Version 3.0 adds five enhanced analysis layers: HTTP response analysis (CSP directive quality, CORS misconfiguration, dangerous HTTP methods, security.txt), HTML content parsing (CSRF tokens, mixed content, vulnerable library fingerprinting, comment disclosures), DNS security depth (SPF/DMARC policy quality, CAA records, DKIM, subdomain takeover detection), TLS certificate depth (cipher suite quality, expiry countdown, self-signed and SAN checks), and path and exposure probing (source maps, backups, dev files, CMS and DB panels, directory listing, error stack traces).

What is Cross-Reference Intelligence?

Cross-Reference Intelligence is the unique differentiator of Canopy Guard. It maps visibility findings against security findings to surface gaps that only exist in the overlap between two disciplines that traditionally never communicate. These are not scored numerically. They are qualitative findings surfaced only when two conditions from different layers combine to create a problem neither layer would flag independently.

  • AI Crawl Gap (Critical): A permissive robots.txt combined with no llms.txt file means AI crawlers like GPTBot, Anthropic, Google-Extended, CCBot, and ByteSpider can scrape your content freely without any citation guidance. Your expertise gets summarized in AI answers but your brand never gets linked.
  • Exposed Endpoints (Critical): Internal API routes accessible in frontend JavaScript bundles can be indexed by AI RAG parsers that scrape JavaScript assets.
  • Content Provenance Risk (Warning): No author or publisher attribution markup allows AI training sets to ingest proprietary material without linking back to the source.

How accurate are the scores?

100 is hard to achieve by design. A perfect SEO score requires fast response time, 1500 or more words of content, 20 or more internal links, exactly one H1, properly lengthed meta description and title, viewport and lang tags, good image alt text coverage, a sitemap, and multiple H2 headings. The scoring methodology is published openly. Every signal and every weight is documented. The tool was validated by scanning a production site that initially scored 11 out of 100, fixing every issue the tool flagged, and watching the score climb to 93 as each fix was deployed.
